## What Is a Custom GPT?

A Custom GPT is a version of ChatGPT configured with:

1. **A system prompt** — Instructions that define the agent's role, tone, and behavior
2. **Knowledge files** — Documents the agent references (company policies, style guides, process docs)
3. **Capabilities** — Web browsing, image generation, or code execution
4. **A name and persona** — Optional, but helps adoption within a team

Once built, the agent is available in the GPT Store (private or shared) and behaves consistently every time it is used.

## Exercise: Build Your Custom GPT (45 min)

### Step 1: Define Your Use Case (5 min)
Choose a use case from your AI Opportunity Canvas. Examples:
- "Board Report Drafter" — helps prepare quarterly board materials
- "Competitive Intelligence Analyst" — summarizes competitor news and implications
- "Meeting Prep Assistant" — creates briefing docs from minimal input
- "RFP Response Helper" — drafts proposal sections from templates
- "HR Policy Advisor" — answers team questions about company policies

### Step 2: Write Your System Prompt (15 min)
Use this template:
```
You are [name], a specialized AI assistant for [company type/function].

Your primary job is to [main task].

You should always:
- [Behavior 1 — e.g., ask for context before drafting]
- [Behavior 2 — e.g., flag assumptions you're making]
- [Behavior 3 — e.g., present options, not just one answer]

You should never:
- [Restriction 1 — e.g., make specific financial projections]
- [Restriction 2 — e.g., reference confidential client names]

Tone: [Formal / Direct / Supportive — choose one]
Format preference: [Bullets / Paragraphs / Tables — choose one]
Audience: [Who will use this agent]
```

### Step 3: Upload Knowledge Files (10 min)
Upload 1–3 relevant documents:
- A style guide or communication template
- A process document the agent should follow
- Background context about your team or company (anonymized if needed)

### Step 4: Test and Refine (15 min)
Ask your agent 3 real questions. Adjust the system prompt based on the outputs until it behaves as intended.

## Facilitator Notes

- The hands-on build is the most high-energy session — budget time generously and let participants experiment
- System prompt quality varies widely — circulate and help the executives who are writing vague instructions
- The showcase is motivating: seeing peers' agents sparks ideas about new use cases
- Common mistake: uploading documents with confidential data. Remind participants to use sanitized or generic documents
- For executives who cannot think of a use case, suggest: "Build a meeting prep assistant — it works for every executive role"
- Have 2–3 pre-built example agents ready to show if the demo lags
